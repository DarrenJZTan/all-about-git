The GitHub Workflow

  1. Fork the project on GitHub
  2. Clone the GitHub fork on to your local computer
  3. Create a branch in your local clone to work on
  4. Make your changes
  5. Commit the changes, remembering to add a commit message
  6. Push the changes back to your GitHub fork
  7. Create a pull request back to the original repository


Commit messages
  • Keep the message short and to the point
  • Use present language (add instead of added, merge instead of merged)
  • Don't end the message with period
  • Do explain what the commit does or doesn't do
